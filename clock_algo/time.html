How It Works

  • ->Pages are arranged in a circular list of memory frames.
  • ->Each frame stores a reference bit that records recent use.
  • ->If the pointer finds a referenced page, it clears the bit and gives that page a second chance.
  • ->The first frame found with reference bit 0 is replaced.

Detailed Explanation

Algorithm Clock / Second Chance

A practical approximation of LRU using a circular pointer and one reference bit per page.

Advantage Low Overhead

The operating system tracks only one reference bit for each frame instead of maintaining a full history.

Tradeoff Not Always Optimal

A frequently used page can remain longer, but the decision is less precise than true least-recently-used replacement.
