Sequential Allocation

Theory

In Sequential Allocation, each file occupies a set of contiguous blocks on the disk. This method is simple to implement and provides excellent performance for sequential access.

  • Simple: Easy to implement and understand.
  • Fast Sequential Access: Reading blocks sequentially is very efficient.
  • Direct Access: Possible to directly access any block if the starting address and length are known.

Disadvantages

  • External Fragmentation: As files are created and deleted, free disk space becomes fragmented into many small, non-contiguous pieces, making it difficult to allocate large new files.
  • Fixed Size: Files cannot grow dynamically beyond their initial allocated space without being moved to a larger contiguous block, which is costly.

Indexed Allocation

Theory

Indexed Allocation solves the external fragmentation problem of sequential allocation. Each file has an index block, which is a dedicated block that stores pointers to all the data blocks belonging to that file. Data blocks can be scattered anywhere on the disk.

  • Eliminates External Fragmentation: Free space can be scattered.
  • Direct Access: Any block can be accessed directly by looking up its pointer in the index block.
  • Dynamic File Size: Files can grow dynamically, limited only by the size of the index block and available free blocks.

Disadvantages

  • Overhead of Index Block: Each file requires at least one entire block for the index, even for small files.
  • Limited File Size: The maximum file size is determined by the number of pointers an index block can hold. Large files might require multiple levels of indexing.

Linked Allocation

Theory

In Linked Allocation, each file is a linked list of disk blocks. Each block contains a pointer to the next block in the chain. The directory entry for a file stores the address of the first and last blocks.

  • No External Fragmentation: Any free block can be used, regardless of its location.
  • Dynamic File Size: Files can grow as long as there are free blocks available.

Disadvantages

  • Poor Random Access Performance: To access a block, all preceding blocks must be read to follow the chain of pointers.
  • Space Overhead: Each data block must reserve space for a pointer to the next block, slightly reducing storage efficiency.
  • Reliability Issues: A single corrupted pointer can lead to the loss of the rest of the file.

Single-Level Directory

Theory

The Single-Level Directory is the simplest directory structure. All files are contained in a single directory, which is shared by all users. This is suitable for single-user systems or small, simple applications.

  • Simplicity: Easy to implement and manage.
  • Fast Access: Locating files is straightforward as there's only one place to look.

Disadvantages

  • Naming Conflicts: All files must have unique names, which becomes problematic with multiple users or many files.
  • Lack of Organization: Difficult to group related files, leading to clutter and poor user experience.
  • No Access Control: All users have access to all files.

Two-Level Directory

Theory

In a Two-Level Directory structure, each user has their own separate directory. There is a master file directory (MFD) that contains entries for each user directory, and each user directory contains entries for their files. This solves the naming conflict problem of the single-level directory.

  • Solves Naming Conflicts: Users can have files with the same name without conflict.
  • User Isolation: Users' files are logically separated, enhancing privacy.

Disadvantages

  • No File Sharing: It is difficult to share files between users without duplicating them.
  • Inefficient for Group Work: Not ideal for collaborative projects where multiple users need to access common files.

Tree-Structured Directory

Theory

The Tree-Structured Directory is the most common directory structure. It allows users to create their own subdirectories, organizing files in a hierarchical (tree-like) manner. Each file and directory has a unique path from the root.

  • Hierarchical Organization: Intuitive and flexible way to organize files.
  • Easy Navigation: Clear paths make it easy to locate files.
  • Solves Naming Conflicts: Files in different branches can have the same name.

Disadvantages

  • Difficult File Sharing: Sharing files across different branches of the tree can be complex.
  • Redundancy: If a file needs to be in multiple categories, it might be duplicated.

Directed Acyclic Graph (DAG) Structure

Theory

A Directed Acyclic Graph (DAG) Structure is an extension of the tree-structured directory that allows files and subdirectories to be shared. This means a file or subdirectory can appear in multiple directories, allowing for more flexible organization and collaboration.

  • Flexible File Sharing: Files can be easily shared by linking them into multiple directories.
  • Eliminates Redundancy: Avoids duplicating files when they belong to multiple categories.

Disadvantages

  • Complexity: More complex to implement and manage due to the possibility of multiple links.
  • Deletion Issues: Deleting a file that is linked in multiple directories can be tricky (e.g., dangling pointers).
  • Cycle Detection: Algorithms are needed to ensure no cycles are created, preventing infinite loops during traversal.
