How It Works

  • ->The CPU completes the current instruction before responding to a maskable interrupt.
  • ->It saves the current process context such as program counter, status, and registers.
  • ->The CPU switches to kernel mode and jumps to the interrupt service routine.
  • ->After the handler finishes, the saved context is restored and execution resumes.
